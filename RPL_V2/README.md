# RPL_V2: служебные скрипты

## 1) `run.py`
Скрипт подготовки `md_test.json` после конвертации Excel в JSON.

### Что делает
- Запускает `RPL_V2/xlsx/ExcelToJson.py`.
- Берёт последний лист из `RPL_V2/xlsx/Матч Тура 25-26.xlsx`.
- Обновляет `RPL_V2/matchday/md_test.json`:
  - `name` — из аргумента запуска.
  - `url` — `https://amaranth64.github.io/RPL_V2/matchday/<sheet>.json`.
  - `picture` — `https://amaranth64.github.io/RPL_V2/matchday/images/<sheet>.webp`.

### Запуск
```bash
py run "Зенит - Балтика"
```
или
```bash
py run.py "Зенит - Балтика"
```

## 2) `create_quiz.py`
Скрипт генерации листа викторины в `Матч Тура 25-26.xlsx` для пары клубов.

### Что делает
- Принимает 2 русских названия клубов (папки в `RPL_V2/Update_Jul_2025`).
- Формирует имя листа `mt_<club1>_<club2>` по `club_*.xlsx`.
- Если лист уже существует — **пересоздаёт** его.
- Добавляет вопросы из файлов клубов:
  - 4 вопроса `Кто этот футболист?`;
  - 4 вопроса других типов;
  - обязательный вопрос про главного тренера (если есть в файле клуба).
- Добавляет матчевые вопросы по Soccer365:
  - прошлый и следующий тур **относительно очного тура введённой пары** (например, для 20-го тура берутся 19-й и 21-й),
  - время следующего матча корректируется на `+1 час`,
  - для вопросов `прошлый/следующий тур` неправильные варианты не повторяются между этими двумя вопросами для одной команды.
- Добавляет турнирные вопросы:
  - лидер РПЛ после **последнего полностью завершённого тура**,
  - место каждого из двух клубов (неправильные места выбираются из диапазона `±4` от реального),
  - лучший бомбардир РПЛ (вопрос пропускается, если у лидеров одинаковое число голов),
  - какая из команд набрала больше очков в последних 5 матчах (вопрос пропускается при равенстве).
- Добавляет вопросы по личным встречам (`&tab=stats_games`):
  - кто чаще побеждал в **официальных** матчах (исключаются только товарищеские),
  - счёт последней встречи (без товарищеских и без кубка),
  - по одному вопросу на каждую сторону: какой игрок забил больше голов в очных встречах за конкретный клуб.
- Кэширует подробную аналитику очных матчей в `RPL_V2/matchday/h2h_cache.json`, чтобы не перепарсивать старые матчи при каждом запуске.
- Формирует подробные логи этапов в консоль.

### Запуск
```bash
py create_quiz "ЦСКА" "Динамо Москва"
```
или
```bash
py create_quiz.py "ЦСКА" "Динамо Москва"
```

### Примечания
- Для парсинга нужен доступ в интернет.
- Скрипт ожидает, что структура и селекторы Soccer365 остаются совместимыми с текущей разметкой.
